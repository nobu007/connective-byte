# ConnectiveByte 公開思想ガイド

ConnectiveByteがpublic repositoryで公開するのは、再利用できる思想、公開可能な設計原則、そして外部に説明責任を持てる判断基準です。

## 公開する思想

### 1. 接続可能性を価値にする

AI時代に重要なのは、個人がどれだけ多く知っているかだけではありません。思考の流れ、判断の理由、前提条件を他者とAIに引き継げる形で残せるかが、再現性と拡張性を決めます。

### 2. 検証できる主張だけを強く言う

ConnectiveByteは、印象論だけで価値を語りません。主張には根拠、判断には条件、改善には計測可能な指標を添えることを重視します。

### 3. AI協創を教育の問題として扱う

AI活用は、ツール選定だけでは終わりません。問いの立て方、レビューの仕方、共有の形式、失敗からの学び方まで含めた教育設計が必要です。

## 公開しないもの

- 内部プロンプト
- 生成ロジックの詳細
- 未公開の実験ログ
- 競争優位に直結する運用ノウハウ

## public repoで伝えること

- なぜその思想を採るのか
- どの技術原則を公開できるのか
- publicとprivateの境界をどう守るのか

このpublic repositoryは、思想の紹介と技術原則の共有を担います。実際の生成ロジック、内部ワークフロー、機密プロンプトはprivate側の connective_byte_core に保持されます。
