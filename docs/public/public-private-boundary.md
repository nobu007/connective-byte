# ConnectiveByte の Public / Private 境界

ConnectiveByteでは、公開すべき知識と秘匿すべき運用資産を明確に分けます。

## public repo に置くもの

- 公開可能な思想の説明
- 再利用できる技術原則
- サイト表示に必要な静的コンテンツ
- 外部に共有して問題ない構成情報

## private repo に置くもの

- 生成ロジック
- 機密プロンプト
- 未公開の設計仕様
- 内部レビュー基準と改善フロー
- 競争優位に直結する検証ノウハウ

## 運用ルール

1. publicへ反映するファイルはmanifestで明示する
2. manifestに載っていないファイルは書き出さない
3. public化した文章は、思想と技術原則に限定する
4. プロンプトや生成器はprivate repoから出さない

この境界を守ることで、公開価値と運用安全性を両立させます。
